# Public Repository Checklist

- [x] Clear README with project purpose, workflow, figures, and quick start
- [x] Source code for training and threshold tuning
- [x] QuPath scripts for upstream tile export
- [x] Lightweight prediction outputs and summary CSVs
- [x] Publication-style figures
- [x] Representative example tiles
- [x] Data availability statement
- [x] Model artifact policy
- [x] Citation metadata
- [x] License
- [x] `.gitignore` excluding large data, checkpoints, and credentials
- [x] Model card
- [x] Experiment report
- [x] CI repository validation
- [x] Resume-positioning notes
- [x] QuPath workflow documentation
- [x] Storage cleanup plan
- [x] Headline accuracy and balanced-accuracy summaries

Before creating an archival release, consider adding:

- DOI through Zenodo
- manuscript citation once available
- GitHub Release with selected model checkpoints, if sharing is approved
- exact commit hash in any manuscript or poster
